# PDPL Compliance Mapping — Agent42

> **UAE Federal Decree-Law No. 45 of 2021** on the Protection of Personal Data ("PDPL")
> Article-by-article mapping of how Agent42 satisfies each provision.

This document is the federal-impact deliverable. It demonstrates that Agent42 is not just an
AI demo but a system architected from day one for PDPL compliance — the difference between a
hackathon toy and a production federal pilot.

| PDPL Article | Provision (paraphrased) | Agent42's implementation |
| --- | --- | --- |
| Art. 4 — Consent | Personal data processing requires consent. | Consent screen shown before any data is collected; consent record persisted with timestamp, scope, and version (audit_log row, kind=`consent`). Right to withdraw exposed in the citizen UI. |
| Art. 5 — Lawful processing | Limited to declared purposes. | Each tool call is logged with its purpose (e.g. `szhp_rules_engine: rescheduling assessment`). Data minimization built into the worker — only the parameters the rules engine needs are extracted from messages. |
| Art. 6 — Cross-border transfer | Restricted unless adequate protection. | Production data residency: **Microsoft Azure UAE North / Core42 / G42 Cloud**. The platform is fully containerised, so it can be deployed on-prem on UAE soil; LLM inference can move to a UAE-hosted/on-prem model when residency requires it. |
| Art. 7 — Data subject rights | Right to access, correct, delete, object. | `/copilot/sessions/{user_id}/transcript` exposes everything we hold; `DELETE` endpoint on roadmap for Day 90. Mem0 supports per-user wipe. |
| Art. 8 — Data security | Appropriate technical and organisational measures. | TLS everywhere; secrets in env (HashiCorp Vault in pilot); audit_log immutable; PII redacted at gateway and again at composer; prompt-injection detection at gateway. |
| Art. 9 — Breach notification | Controller notifies the Data Office without undue delay. | Langfuse alerting webhook → Customer Happiness Centre SOC. Breach SOP in pilot RACI. |
| Art. 12 — DPO | Designate a Data Protection Officer for high-risk processing. | Pilot RACI names the MOEI DPO for the SZHP integration. |
| Art. 13 — Cross-system processing | Joint controllers must define responsibilities. | UAE PASS identity verification flows are jointly governed by MOEI and TDRA; agreement modelled in the 90-day pilot. |

## Right to explanation

Every supervisor run produces a Langfuse trace with full inputs/outputs at every node.
The citizen-facing audit-trail UI lets any citizen (or auditor) click a decision and see:

- which Router intent/service was selected and why (reasoning field on the structured output)
- which tools were called with which arguments (HousingAgent → `szhp_rules_engine`, `risk_score`)
- the rule citations that produced the recommendation (e.g. "SZHP-R3.1: Arrears ≥ 3 months — eligible for hardship rescheduling")
- the critic verdict and whether the draft was re-planned
- whether the case was escalated to a human, and the reason

This is what makes the rules engine architectural choice load-bearing: **policy is not in the
LLM, it is in deterministic Python that the agent calls**. Explanation is therefore exact, not
post-hoc rationalisation.

## Data minimization

- Citizen messages are PII-redacted before any external LLM call when configured to log inputs.
- Only the case parameters the rules engine needs (months_in_arrears, income, balance, employment) are extracted; raw documents are kept only for audit.
- Mem0 entries are scoped by `user_id` and never cross-referenced across citizens.

## Synthetic demo data

All data used in the hackathon demo is **synthetic**, generated by `scripts/gen_synthetic_data.py`:
- 300 rescheduling cases with Faker-generated Emirati names + valid-format-but-not-real Emirates IDs (last digit is a placeholder)
- 100 salary slips with synthetic employer + amounts
- The mapping from a WhatsApp number to Emirates ID in `whatsapp.py` is hardcoded to one synthetic identity

The pilot will replace this with the UAE PASS sandbox; under PDPL Art. 5 we may only process
real data once the production consent flow is signed off by MOEI's Customer Happiness Centre.
